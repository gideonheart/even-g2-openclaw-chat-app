<objective>
Add a FIFO voice turn queue to VoiceLoopController so multiple recordings execute sequentially without aborting in-flight gateway requests.

Purpose: The gateway client calls abort() on every sendVoiceTurn invocation, cancelling any in-flight fetch. Without a queue, starting a second recording while the first is still processing causes the first turn's response to be aborted. The queue holds pending turns and drains one at a time when the gateway signals completion.

Output: Modified voice-loop-controller.ts with queue logic, comprehensive queue tests.
</objective>

```typescript
// Voice turn queue -- ensures sequential execution, no abort of in-flight requests.
// Gateway client calls abort() on every sendVoiceTurn, so concurrent sends would cancel
// the in-flight turn. The queue holds pending turns and drains one at a time.
const MAX_QUEUE = 5;
interface PendingTurn { sessionId: string; blob: Blob }
const pendingTurns: PendingTurn[] = [];
let busy = false;

function processQueue(): void {
  if (busy || pendingTurns.length === 0) return;
  busy = true;
  const turn = pendingTurns.shift()!;
  const s = settings();
  bus.emit('log', {
    level: 'info',
    msg: `Sending voice turn: ${turn.blob.size} bytes ${turn.blob.type} to ${s.gatewayUrl || '(not set)'} stt=${s.sttProvider} (queue: ${pendingTurns.length} remaining)`,
  });
  gateway.sendVoiceTurn(s, {
    sessionId: turn.sessionId,
    audio: turn.blob,
    sttProvider: s.sttProvider,
  });
}
```

Modify the existing `gateway.onChunk` subscription to also drain the queue on response_end/error. Add these lines AFTER `bus.emit('gateway:chunk', chunk)`:

```typescript
// Drain queue on response_end or error (turn lifecycle is complete)
if (chunk.type === 'response_end' || chunk.type === 'error') {
  busy = false;
  processQueue();
}
```

Replace the `audio:recording-stop` handler (currently immediate send) with enqueue logic:

```typescript
// When recording stops, enqueue voice turn (sequential, not immediate send)
unsubs.push(bus.on('audio:recording-stop', ({ sessionId, blob }) => {
  if (pendingTurns.length >= MAX_QUEUE) {
    bus.emit('log', { level: 'warn', msg: `Voice queue full (${MAX_QUEUE}), dropping oldest turn` });
    pendingTurns.shift();
  }
  pendingTurns.push({ sessionId, blob });
  processQueue();
}));
```

Update the `destroy()` function to clear queue state:

```typescript
function destroy(): void {
  for (const unsub of unsubs) {
    unsub();
  }
  unsubs.length = 0;
  pendingTurns.length = 0;
  busy = false;
}
```

Also add `getQueueLength` to the returned interface for testability:

```typescript
return { destroy, getQueueLength: () => pendingTurns.length };
```

Update the `VoiceLoopController` interface to include the new method:

```typescript
export interface VoiceLoopController {
  destroy(): void;
  getQueueLength(): number;
}
```

Key design constraints (from research):
- Do NOT make the queue async/Promise-based. Gateway client signals completion via chunks (synchronous bus dispatch), not Promises.
- Do NOT modify gateway-client.ts. The abort() in sendVoiceTurn is harmless when called sequentially.
- Do NOT add turn ID generation or duplicate guard. The research recommends it but it adds complexity for a race condition that cannot occur with synchronous dispatch. Keep it simple -- if needed later, it's trivial to add.
- The `busy` boolean is correct because the event bus is synchronous and single-threaded.

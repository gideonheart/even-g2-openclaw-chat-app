**Step 1: Create `src/gestures/gesture-handler.ts`**

Follow RESEARCH.md Pattern 3. Factory function:

```typescript
createGestureHandler(opts: {
  bus: EventBus<AppEventMap>;
  bridge: BridgeService;
  audioCapture: AudioCapture;
  activeSessionId: () => string;
}): GestureHandlerAPI
```

Returns `GestureHandlerAPI` with:
- `getState(): GestureState` — current FSM state
- `getHintText(): string` — contextual hint for current state
- `destroy(): void` — unsubscribes from bus

Internal state:
- `state: GestureState = 'idle'`
- `lastTapTs: number = 0`
- `const DEBOUNCE_MS = 275` (middle of 250-300ms range per GEST-05)

`handleInput(input: GestureInput, timestamp: number)`:
1. Debounce check: if `input === 'tap'` and `(timestamp - lastTapTs) < DEBOUNCE_MS`, return (suppress rapid tap after previous tap)
2. If `input === 'tap'`, update `lastTapTs = timestamp`
3. Call `gestureTransition(state, input)` to get `{ nextState, action }`
4. Update `state = nextState`
5. If action is not null, call `dispatchAction(action)`

`dispatchAction(action: GestureAction)`:
- `START_RECORDING`: Call `audioCapture.startRecording(activeSessionId())` → `bridge.startAudio()` → `bus.emit('audio:recording-start', { sessionId: activeSessionId() })`
- `STOP_RECORDING`: Call `bridge.stopAudio()` → `audioCapture.stopRecording()` → `bus.emit('audio:recording-stop', { sessionId: activeSessionId(), blob })` (blob from audioCapture.stopRecording())
- `TOGGLE_MENU`: Emit on bus `'gesture:menu-toggle'` (add to AppEventMap: `'gesture:menu-toggle': { active: boolean }` — active=true if nextState is 'menu')
- `SCROLL_UP` / `SCROLL_DOWN`: These are already on the bus from the bridge. No additional dispatch needed — the handler just advanced the FSM state. Phase 3 display will listen directly.

Wire to bus events:
- `bus.on('gesture:tap', (p) => handleInput('tap', p.timestamp))`
- `bus.on('gesture:double-tap', (p) => handleInput('double-tap', p.timestamp))`
- `bus.on('gesture:scroll-up', (p) => handleInput('scroll-up', p.timestamp))`
- `bus.on('gesture:scroll-down', (p) => handleInput('scroll-down', p.timestamp))`

Store all unsubscribe functions. `destroy()` calls them all.

**Hint bar text generator** — export `getHintText(state: GestureState): string`:
- 'idle': 'Tap to record | Double-tap for menu'
- 'recording': 'Tap to stop recording'
- 'sent': 'Processing...'
- 'thinking': 'AI is thinking... | Double-tap for menu'
- 'menu': 'Double-tap to close | Scroll to navigate'

Add `'gesture:menu-toggle': { active: boolean }` to `AppEventMap` in `src/types.ts`.

**Step 2: Create `src/__tests__/gesture-handler.test.ts`**

Test debounce behavior:
- Two taps 100ms apart: second tap suppressed, only one START_RECORDING emitted
- Two taps 300ms apart: both processed (first START_RECORDING, second STOP_RECORDING)
- Tap then double-tap 200ms later: tap suppressed by debounce, double-tap processed (this is the critical race condition)

Test action dispatch:
- Tap in idle → emits `audio:recording-start` on bus
- Tap in recording → emits `audio:recording-stop` on bus with blob
- Double-tap in idle → emits `gesture:menu-toggle` with active=true

Test state tracking:

Test bridge integration:
- START_RECORDING calls `bridge.startAudio()` and `audioCapture.startRecording()`

Test destroy:

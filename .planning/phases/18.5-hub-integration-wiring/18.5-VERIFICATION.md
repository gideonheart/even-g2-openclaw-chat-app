### Integration Gap Closure Summary

This phase closes all 3 integration gaps and 2 broken E2E flows identified by the v1.3 milestone audit:

**Gap 1 (HIGH): Hub error presenter dead code** -- CLOSED. 5 `hubBus.emit('persistence:error', ...)` calls added across all hub error paths. The presenter now receives events and activates.

**Gap 2 (MEDIUM): Hub stale store after reopenDB** -- CLOSED. `hubDriftReconciler` and `hubSyncMonitor` are now destroyed and recreated with the fresh `newConversationStore` handle in the reopenDB success handler.

**Gap 3 (MEDIUM): Hub storage health dot never updates** -- CLOSED. Boot-time quota is now returned from `initPersistence()`, stored in `cachedQuota`, emitted to the `persistence:health` subscriber, and read by `refreshHealthDisplay()` for subsequent visits.

**E2E Flow 1 (Hub persistence error -> Toast/Banner via error presenter)** -- RESTORED. hubBus now receives events; presenter routes them to toast/banner based on `recoverable` flag.

**E2E Flow 2 (Hub boot -> Storage health dot updated)** -- RESTORED. Quota emitted after subscriber registration; `refreshHealthDisplay` uses `cachedQuota` for on-demand updates.

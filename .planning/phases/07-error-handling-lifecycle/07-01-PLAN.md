<objective>
Add error recovery to the voice loop so gateway failures and timeouts return the user to idle state with visible feedback.

Purpose: Without error recovery, a gateway failure leaves the gesture FSM stuck in `sent` or `thinking` -- the user must restart the app. This plan adds a `reset` FSM input, a 30-second request timeout, and user-visible error messages so the user can immediately retry after any failure.

Output: Modified gesture FSM, gesture handler, gateway client, display controller, and glasses renderer with full test coverage.
</objective>

- Add `'reset'` to the `GestureInput` union type: `export type GestureInput = 'tap' | 'double-tap' | 'scroll-up' | 'scroll-down' | 'reset';`
- Add `reset` transition to every state in the TRANSITIONS table:
  - `idle`: `'reset': { nextState: 'idle', action: null }` (no-op, already idle)
  - `recording`: `'reset': { nextState: 'idle', action: { type: 'STOP_RECORDING' } }` (stop recording before resetting)
  - `sent`: `'reset': { nextState: 'idle', action: null }`
  - `thinking`: `'reset': { nextState: 'idle', action: null }`
  - `menu`: `'reset': { nextState: 'idle', action: null }`

**2. Add error listener to gesture handler (`src/gestures/gesture-handler.ts`):**

- In `createGestureHandler`, after the existing bus subscriptions for gesture events, add a subscription to `gateway:chunk` that checks for error chunks:
  ```typescript
  unsubs.push(bus.on('gateway:chunk', (chunk) => {
    if (chunk.type === 'error') {
      handleInput('reset', Date.now());
    }
  }));
  ```
- The `reset` input bypasses the 275ms tap debounce (debounce only applies to `tap` input, which is already the case in the existing code).

**3. Add timeout signal to gateway client (`src/api/gateway-client.ts`):**

- Add a constant: `const TURN_TIMEOUT_MS = 30_000;`
- In `sendVoiceTurn`, after creating the `abortController`, compose a combined signal:
  ```typescript
  const combinedSignal = AbortSignal.any([
    abortController.signal,
    AbortSignal.timeout(TURN_TIMEOUT_MS),
  ]);
  ```
- Pass `signal: combinedSignal` to the `fetch` call instead of `abortController.signal`.
- In the `catch` block, add a `TimeoutError` check BEFORE the existing retry logic:
  ```typescript
  if (err instanceof DOMException && err.name === 'TimeoutError') {
    emitChunk({ type: 'error', error: 'Request timed out. Tap to retry.' });
    setStatus('error');
    return; // No retry on timeout
  }
  ```
  This MUST be placed after the existing `AbortError` check and BEFORE the `reconnectAttempts` retry logic. Timeout errors must NOT trigger exponential backoff.

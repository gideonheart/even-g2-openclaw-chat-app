function countMessages(conversationId: string): Promise<number> {

**GREEN phase: Implement src/sync/sync-monitor.ts:**

```typescript
export function createSyncMonitor(opts: {
  bridge: SyncBridge;
  store: { countMessages(conversationId: string): Promise<number> };
  origin: SyncOrigin;
  getActiveConversationId: () => string;
  onHeartbeat?: (conversationId: string, remoteCount: number) => void;
}): SyncMonitor
```

Implementation details:
- Maintain `localSeq = 0`, `lastRemoteSeq = -1`, `lastReceivedAt = 0`, `sequenceGaps = 0`.
- `send(msg)`: increment localSeq, call `bridge.postMessage({ ...msg, seq: localSeq })`.
- Subscribe to bridge.onMessage: filter own origin, track lastReceivedAt = Date.now(), handle seq tracking:
  - If `msg.seq !== undefined`:
    - If `lastRemoteSeq >= 0 && msg.seq > lastRemoteSeq + 1`: gap detected, add `msg.seq - lastRemoteSeq - 1` to sequenceGaps.
    - If `lastRemoteSeq >= 0 && msg.seq <= lastRemoteSeq`: peer reboot, reset lastRemoteSeq to msg.seq, do NOT increase gaps.
    - Set `lastRemoteSeq = msg.seq`.
  - If `msg.type === 'sync:heartbeat'`: call onHeartbeat callback with conversationId and messageCount.
- `startHeartbeat()`: setInterval at 10_000ms. Each tick: get activeConversationId, if empty return. Call store.countMessages, then send sync:heartbeat message.
- `stopHeartbeat()`: clearInterval.
- `destroy()`: stopHeartbeat, call bridge onMessage unsubscribe function.
- `getStats()`: return { localSeq, lastRemoteSeq, sequenceGaps, lastReceivedAt, heartbeatActive }.

Zero new dependencies. Uses existing SyncBridge interface.

Create a test file testing createDriftReconciler with these behaviors:
- Uses a mock ConversationStore (countMessages as vi.fn()).
- Uses a mock EventBus or callback pattern.

Per research Open Question 1: DriftReconciler should use callback pattern (not bus) for portability across glasses (has bus) and hub (no bus). Interface:
```typescript
export function createDriftReconciler(opts: {
  store: { countMessages(conversationId: string): Promise<number> };
  onDriftDetected?: (info: { localCount: number; remoteCount: number; conversationId: string }) => void;
  onReconciled?: (info: { conversationId: string; action: 'full-reload' }) => void;
}): DriftReconciler
```

Test cases:
1. **No drift when counts match** -- store.countMessages returns 5, call handleHeartbeat(convId, 5). Returns false. No callbacks fired.
2. **Single mismatch does NOT trigger reconciliation** -- store returns 5, call handleHeartbeat(convId, 7). Returns false. No onDriftDetected/onReconciled called (2-consecutive rule per research: only trigger after 2 consecutive mismatches).
3. **Two consecutive mismatches trigger reconciliation** -- store returns 5 both times, call handleHeartbeat(convId, 7) twice. Second call returns true, onDriftDetected called with { localCount: 5, remoteCount: 7, conversationId }, onReconciled called with { conversationId, action: 'full-reload' }.
4. **Mismatch followed by match resets counter** -- store returns 5, call handleHeartbeat(convId, 7) [mismatch]. Then store returns 7, call handleHeartbeat(convId, 7) [match]. Counter resets. Then store returns 7, call handleHeartbeat(convId, 9) [mismatch again]. Returns false (first mismatch, not second).
5. **Different conversation ID resets mismatch counter** -- mismatch on convA, then mismatch on convB. Neither triggers reconciliation (conversation changed, counter resets).
6. **Works with zero callbacks** -- handleHeartbeat with no callbacks set does not throw.
7. **destroy clears internal state** -- after destroy, handleHeartbeat with previously-mismatched conversation does not trigger.

**GREEN phase: Implement src/sync/drift-reconciler.ts:**

Implementation:
- Track `consecutiveMismatches = 0`, `lastMismatchConvId = ''`.
- `handleHeartbeat(conversationId, remoteCount)`:
  - `const localCount = await store.countMessages(conversationId)`.
  - If localCount === remoteCount: reset consecutiveMismatches to 0, return false.
  - If conversationId !== lastMismatchConvId: reset consecutiveMismatches to 0, set lastMismatchConvId.
  - Increment consecutiveMismatches.
  - If consecutiveMismatches >= 2: call onDriftDetected, call onReconciled with action 'full-reload', reset consecutiveMismatches, return true.
  - Else return false.
- `destroy()`: reset all tracking state.

<objective>
Add gateway error classification to distinguish connection errors (safe to retry) from mid-stream errors (do NOT retry). Prevents duplicate backend requests when streaming fails after data has been received.

Purpose: The gateway client currently retries all errors uniformly. When a stream fails mid-response (after the backend has processed the request and begun sending data), retrying sends the same request again, causing duplicate AI responses. Adding a `receivedAnyData` flag in `streamSSEResponse()` allows `handleTurnError()` to skip retries for mid-stream failures while preserving existing retry logic for connection failures. This directly addresses Pitfall P7 from the project PITFALLS.md.

Output: Modified gateway-client.ts with error classification, extended gateway-client test suite with mid-stream error tests.
</objective>

**Step 1: Modify `streamSSEResponse` signature to accept a `streamState` object.**

Change the function signature from:
```typescript
async function streamSSEResponse(resp: Response, timeoutId: ReturnType<typeof setTimeout>): Promise<void>
```
to:
```typescript
async function streamSSEResponse(
  resp: Response,
  timeoutId: ReturnType<typeof setTimeout>,
  streamState: { receivedAnyData: boolean },
): Promise<void>
```

Inside the function body, after `buffer += decoder.decode(value, { stream: true });`, add:
```typescript
streamState.receivedAnyData = true;
```

This sets the flag after the first successful `reader.read()` that yields bytes. The flag is visible in the catch block via the shared object reference.

**Step 2: Modify `handleTurnError` to accept `receivedAnyData` parameter and return 'mid-stream' type.**

Change the return type from `'abort' | 'timeout' | 'retry' | 'fatal'` to `'abort' | 'timeout' | 'retry' | 'mid-stream' | 'fatal'`.

Add a third parameter:
```typescript
function handleTurnError(
  err: unknown,
  timeoutId: ReturnType<typeof setTimeout>,
  receivedAnyData: boolean = false,
): 'abort' | 'timeout' | 'retry' | 'mid-stream' | 'fatal' {
```

After the existing TimeoutError handling block and before the existing connection error logic (`const message = ...`), add the mid-stream check:
```typescript
// Mid-stream failure: data was being received, do NOT retry (Pitfall P7)
if (receivedAnyData) {
  emitChunk({ type: 'error', error: 'Response interrupted \u2014 tap to ask again' });
  setStatus('error');
  return 'mid-stream';
}
```

The existing connection error logic (after this new block) remains unchanged.

**Step 3: Modify `sendVoiceTurn` to use `streamState` pattern.**

Before the `try` block, add:
```typescript
const streamState = { receivedAnyData: false };
```

Pass `streamState` to `streamSSEResponse`:
```typescript
await streamSSEResponse(resp, timeoutId, streamState);
```

In the `catch` block, pass `streamState.receivedAnyData` to `handleTurnError`:
```typescript
const result = handleTurnError(err, timeoutId, streamState.receivedAnyData);
```

No change to the retry logic -- `'mid-stream'` result falls through (no `if (result === 'retry')` match, so no retry occurs). The error chunk with "Response interrupted" message was already emitted by `handleTurnError`.

**Step 4: Apply identical changes to `sendTextTurn`.**

Same pattern as sendVoiceTurn:
1. Add `const streamState = { receivedAnyData: false };` before try block
2. Pass `streamState` to `streamSSEResponse`
3. Pass `streamState.receivedAnyData` to `handleTurnError` in catch block

**Key invariants:**
- The `streamState` object is shared between the try/catch scopes via closure. When `streamSSEResponse` throws, `streamState.receivedAnyData` reflects whether any bytes were read before the error.
- Mid-stream errors emit an error chunk with "Response interrupted" message (distinct from connection errors).
- Mid-stream errors do NOT increment `reconnectAttempts` (not relevant for future retries).
- The `receivedAnyData` parameter defaults to `false` in `handleTurnError`, preserving backward compatibility.
- Existing abort, timeout, and connection error handling is completely unchanged.

describe('mid-stream error classification', () => {
  it('does NOT retry when reader throws after receiving data', async () => {
    let readCount = 0;
    const encoder = new TextEncoder();
    globalThis.fetch = vi.fn().mockResolvedValue({
      ok: true,
      status: 200,
      statusText: 'OK',
      body: new ReadableStream({
        pull(controller) {
          readCount++;
          if (readCount === 1) {
            controller.enqueue(encoder.encode('data: {"type":"response_delta","text":"hi"}\n\n'));
          } else {
            controller.error(new Error('Connection reset'));
          }
        },
      }),
    });

    const client = createGatewayClient({
      maxReconnectAttempts: 3,
      reconnectBaseDelayMs: 1,
    });
    const chunks: VoiceTurnChunk[] = [];
    client.onChunk((c) => chunks.push(c));

    await client.sendVoiceTurn(testSettings, testRequest);

    // Should emit the delta chunk + the mid-stream error chunk
    expect(chunks.some((c) => c.type === 'response_delta')).toBe(true);
    const errorChunks = chunks.filter((c) => c.type === 'error');
    expect(errorChunks).toHaveLength(1);
    expect(errorChunks[0].error).toContain('interrupted');

    // fetch called only once -- no retry
    expect(globalThis.fetch).toHaveBeenCalledTimes(1);
  });

  it('sets status to error on mid-stream failure', async () => {
    let readCount = 0;
    const encoder = new TextEncoder();
    globalThis.fetch = vi.fn().mockResolvedValue({
      ok: true,
      status: 200,
      statusText: 'OK',
      body: new ReadableStream({
        pull(controller) {
          readCount++;
          if (readCount === 1) {
            controller.enqueue(encoder.encode('data: {"type":"response_start"}\n\n'));
          } else {
            controller.error(new Error('Stream broken'));
          }
        },
      }),
    });

    const client = createGatewayClient({ reconnectBaseDelayMs: 1 });
    const statuses: string[] = [];
    client.onStatusChange((s) => statuses.push(s));

    await client.sendVoiceTurn(testSettings, testRequest);

    // Final status should be error
    expect(statuses[statuses.length - 1]).toBe('error');
  });

  it('does NOT increment reconnectAttempts on mid-stream failure', async () => {
    let readCount = 0;
    const encoder = new TextEncoder();
    globalThis.fetch = vi.fn().mockResolvedValue({
      ok: true,
      status: 200,
      statusText: 'OK',
      body: new ReadableStream({
        pull(controller) {
          readCount++;
          if (readCount === 1) {
            controller.enqueue(encoder.encode('data: {"type":"response_delta","text":"data"}\n\n'));
          } else {
            controller.error(new Error('Connection lost'));
          }
        },
      }),
    });

    const client = createGatewayClient({ reconnectBaseDelayMs: 1 });
    await client.sendVoiceTurn(testSettings, testRequest);

    expect(client.getHealth().reconnectAttempts).toBe(0);
  });

  it('still retries when fetch rejects before any response (connection error)', async () => {
    let callCount = 0;
    const sseData = ['data: {"type":"response_end"}\n\n'];
    globalThis.fetch = vi.fn().mockImplementation(() => {
      callCount++;
      if (callCount === 1) return Promise.reject(new Error('Network failure'));
      return Promise.resolve({
        ok: true,
        status: 200,
        statusText: 'OK',
        body: createSSEStream(sseData),
      });
    });

    const client = createGatewayClient({
      maxReconnectAttempts: 3,
      reconnectBaseDelayMs: 1,
    });
    await client.sendVoiceTurn(testSettings, testRequest);

    // Should have retried (2 calls total)

  });
});

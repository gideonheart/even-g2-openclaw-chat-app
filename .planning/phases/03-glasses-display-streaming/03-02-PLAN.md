<task type="auto">
  <name>Task 2: Create GlassesRenderer service with streaming, scroll, and hide/wake</name>
  <files>
    src/display/glasses-renderer.ts
    src/__tests__/glasses-renderer.test.ts
  </files>
  <action>
    Create src/display/glasses-renderer.ts as a factory function:

    ```
    createGlassesRenderer(opts: { bridge: BridgeService; bus: EventBus&lt;AppEventMap&gt; }): GlassesRenderer
    ```

    Export GlassesRenderer interface with these methods:
    - init(): Promise&lt;void&gt; -- calls bridge.rebuildPageContainer with 3-container layout (see below), starts icon animator, sets icon to 'idle'
    - destroy(): void -- stops icon animator, stops stream flush timer, clears state
    - setIconState(state: IconState): void -- delegates to icon animator setState, updates status text container via textContainerUpgrade(containerID=1, iconText)
    - addUserMessage(text: string): void -- appends ChatMessage with role='user' to viewport messages, re-renders viewport if autoScroll
    - startStreaming(): void -- appends empty ChatMessage with role='assistant', complete=false. Starts 200ms flush interval (CHAT-03)
    - appendStreamChunk(text: string): void -- accumulates in streamBuffer (not directly appended). The flush interval handles batched rendering.
    - endStreaming(): void -- stops flush interval, final flush of remaining buffer, marks current assistant message complete=true, re-renders
    - scrollUp(): void -- calls viewport scrollUp, re-renders chat container
    - scrollDown(): void -- calls viewport scrollDown, re-renders chat container
    - hide(): Promise&lt;void&gt; -- stops icon animator, calls rebuildPageContainer with blank layout, sets hidden=true
    - wake(): Promise&lt;void&gt; -- calls rebuildPageContainer to restore 3-container layout, restarts icon animator, re-renders current state, sets hidden=false
    - isHidden(): boolean
    - getHintText(): string -- delegates to gesture handler's getHintText (passed via bus event or direct reference)
    - updateHint(text: string): void -- updates hint text container via textContainerUpgrade(containerID=3, text)

    3-container text layout constants (per research, using text containers only -- no image containers):
    - Container 1 (status): xPosition=0, yPosition=0, width=576, height=30, containerID=1, containerName='status', isEventCapture=0
    - Container 2 (chat): xPosition=0, yPosition=34, width=576, height=224, containerID=2, containerName='chat', isEventCapture=0
    - Container 3 (hint): xPosition=0, yPosition=260, width=576, height=28, containerID=3, containerName='hint', isEventCapture=0

    NOTE: Set isEventCapture=0 on ALL containers. We manage scrolling in JavaScript (per research: firmware scroll has no position API). This avoids the firmware consuming scroll gestures internally.

    Blank layout for hide mode:
    - containerTotalNum=1, single text container: full-screen, empty content, containerName='blank', isEventCapture=0

    Internal state:
    - ViewportState (from viewport.ts): messages array, scrollOffset, autoScroll
    - hidden: boolean
    - streamBuffer: string (accumulated chunks between flushes)
    - flushTimer: ReturnType&lt;typeof setInterval&gt; | null

    Streaming flush loop (200ms interval):
    - If streamBuffer has content, append to current (last) assistant message text, clear buffer, call renderAndPush()
    - renderAndPush(): call renderViewport(state), then bridge.textContainerUpgrade(2, renderedText)

    Tests (src/__tests__/glasses-renderer.test.ts):
    Use a mock BridgeService (not the SDK mock -- a simple vi.fn() stub implementing BridgeService interface).

    Test cases:
    - init() calls rebuildPageContainer with 3-container layout
    - addUserMessage appends message and renders viewport to chat container
    - startStreaming + appendStreamChunk + wait 200ms -> flushes to display
    - endStreaming does final flush and marks message complete
    - scrollUp/scrollDown adjust viewport and re-render
    - hide() calls rebuildPageContainer with blank layout
    - wake() calls rebuildPageContainer with 3-container layout and re-renders
    - setIconState updates status container text
    - No textContainerUpgrade call exceeds 2000 chars (CHAT-07)
  </action>

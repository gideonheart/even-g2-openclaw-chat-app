    ## Viewport (viewport.ts)

    Types:
    - ChatMessage: { id: string; role: 'user' | 'assistant'; text: string; complete: boolean; timestamp: number }
    - ViewportState: { messages: ChatMessage[]; scrollOffset: number; autoScroll: boolean }

    Pure functions:
    - serializeMessages(messages: ChatMessage[]): string
      - User messages prefixed with '> ' (CHAT-01 right-alignment simulation)
      - Assistant messages have no prefix (CHAT-02 left-alignment)
      - Messages separated by blank line ('\n\n')
      - Incomplete messages get ' ...' suffix (streaming indicator)

    - renderViewport(state: ViewportState): string
      - scrollOffset=0 means show latest (bottom of history) (CHAT-04 virtualized)
      - Builds text from bottom up, stops at MAX_VIEWPORT_CHARS (1800)
      - Hard truncation at MAX_VIEWPORT_CHARS to enforce CHAT-07 (< 2000 chars)
      - Empty messages array returns ''

    - scrollUp(state: ViewportState): ViewportState
      - Increments scrollOffset by 1 (CHAT-05)
      - Clamps at messages.length - 1 (can't scroll past first message)
      - Sets autoScroll = false

    - scrollDown(state: ViewportState): ViewportState
      - Decrements scrollOffset by 1 (CHAT-06)
      - Clamps at 0 (can't scroll past latest)
      - When scrollOffset reaches 0, sets autoScroll = true

    Test cases:
    - serializeMessages([]) -> ''
    - serializeMessages([{role:'user', text:'hello', complete:true}]) -> '> hello'
    - serializeMessages([{role:'assistant', text:'hi', complete:true}]) -> 'hi'
    - serializeMessages([user msg, assistant msg]) -> '> hello\n\nhi'
    - serializeMessages with incomplete assistant -> 'hi ...'
    - renderViewport with empty messages -> ''
    - renderViewport with scrollOffset=0 shows last message
    - renderViewport truncates at MAX_VIEWPORT_CHARS
    - renderViewport with scrollOffset>0 shows earlier messages
    - scrollUp from offset 0 -> offset 1, autoScroll false
    - scrollUp at max -> stays at max
    - scrollDown from offset 1 -> offset 0, autoScroll true
    - scrollDown at 0 -> stays at 0

    ## Icon Animator (icon-animator.ts)

    Types:
    - IconState: 'idle' | 'recording' | 'sent' | 'thinking'

    Factory: createIconAnimator(updateFn: (text: string) => Promise<unknown>)
    - setState(state: IconState): void -- switches state, resets frameIdx, calls immediate tick
    - start(): void -- begins setInterval at 200ms (~5fps)
    - stop(): void -- clears interval
    - getState(): IconState
    - Frame-drop: if previous updateFn hasn't resolved, skip tick (set inFlight flag)

    ## Icon Bitmaps (icon-bitmaps.ts) -- Unicode text approach

    Per research Open Question #3 (Claude's Discretion): use Unicode text icons instead of image containers for v1. This eliminates the 4-bit pixel packing ambiguity and reduces container count.

    ICON_FRAMES: Record<IconState, string[]>
    - idle: ['◌'] -- single static frame (open circle)
    - recording: ['●', '○'] -- 2-frame blink (filled/empty circle)
    - sent: ['✓'] -- single static frame (checkmark)
    - thinking: ['⠋', '⠙', '⠹', '⠸', '⠼', '⠴', '⠦', '⠧', '⠇', '⠏'] -- braille spinner (10 frames)

    Test cases for icon-animator:
    - setState('idle') calls updateFn with '◌' immediately
    - setState('recording') calls updateFn with '●' immediately
    - start() calls updateFn on 200ms interval
    - stop() clears interval (no more calls)
    - Frame drop: if updateFn is slow (unresolved promise), next tick skips
    - setState resets frameIdx (cycling starts from frame 0)

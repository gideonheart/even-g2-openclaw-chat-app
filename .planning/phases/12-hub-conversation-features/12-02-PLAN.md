   - Import `createGatewayClient` from `./api/gateway-client`
   - Create a gateway client instance: `hubGateway = createGatewayClient()`
   - Wire the gateway client's `onChunk` handler to process response chunks:
     - On `response_start`: call `showStreamingIndicator()`. Also post `streaming:start` sync message via `hubSyncBridge` to notify glasses.
     - On `response_delta`: accumulate text in a module-level `pendingHubAssistantText` variable. On `response_delta`, update that div's textContent by appending chunk.text. On `response_end`, finalize.
     - On `response_end`: hide streaming indicator. Save the full assistant text to IndexedDB via `hubConversationStore.addMessage()`. Post `message:added` sync message (origin: 'hub', role: 'assistant') and `streaming:end` sync message. Clear `pendingHubAssistantText`.
     - On `error`: hide streaming indicator. Post `streaming:end` sync message. Show error as toast. Clear pending text.

4. Add `handleTextSubmit(text: string)` async function:
   - Get active session ID from `sessionManager.getActiveSessionId()`. If none, show toast "No active session" and return.
   - Save user message to IndexedDB: `await hubConversationStore.addMessage(activeId, { role: 'user', text, timestamp: Date.now() })`
   - Post `message:added` sync message (origin: 'hub', role: 'user', text) to notify glasses
   - Call `appendLiveMessage('user', text)` to show in hub live view
   - Call `hubGateway.sendTextTurn(appState.settings, { sessionId: activeId, text })`

5. In `init()`, wire the text form submit event:
   - `$('hubTextForm').addEventListener('submit', (e) => { e.preventDefault(); ... })`
   - Get value from `$('hubTextInput')`, trim it. If empty, return.
   - Clear the input field.
   - Disable the send button during the request (re-enable on response_end or error).
   - Call `handleTextSubmit(value)`.

6. Add cleanup for `hubGateway` in the `beforeunload` handler: `hubGateway?.destroy()`.

7. For the streaming response display approach: Instead of calling appendLiveMessage for each delta, use a streaming approach:
   - On `response_start`: create a new div with class `chat-msg chat-msg--assistant`, empty textContent, append to live view container, store reference as `streamingMsgEl`.
   - On `response_delta`: append `chunk.text` to `streamingMsgEl.textContent`. Auto-scroll.
   - On `response_end`: `streamingMsgEl = null`. The div stays with the full text.

This gives the user real-time streaming text in the hub for messages they type from the hub. For messages from glasses (via sync), they arrive as complete messages since sync only fires on `message:added` (after save).

### Stale db Handle Detail (Medium)

`reopenDB()` returns a new `IDBDatabase` instance but both `glasses-main.ts` and `hub-main.ts` use `.then(() => ...)` (no argument), discarding the fresh handle. All stores (`ConversationStore`, `SessionStore`, `IntegrityChecker`, auto-save) close over the original dead `db` reference. After reconnection, writes go through the dead handle.

**Mitigation:** Phase 18 (Error UX) plans a "restart required" message for `recoverable: false` events, which limits the exposure window. Data loss only occurs if a user saves during the brief window between onclose and app restart.

### Issue 1: Stale db Handle (Critical)

`reopenDB()` returns a new `IDBDatabase` instance but both boot files use `.then(() => ...)` (no argument), discarding the fresh handle. All stores (`ConversationStore`, `SessionStore`, `IntegrityChecker`, auto-save) close over the original dead `db` reference. After reconnection, writes go through the dead handle and fail with `InvalidStateError`.

**Impact:** Every IDB operation after an unexpected close + successful reconnect silently fails. The "reconnected successfully" log message is misleading — only the DB-level connection succeeds; the application layer is broken.

**Mitigation:** Phase 18 (Error UX) plans a "restart required" message for `recoverable: false` events, which limits exposure. However the root cause (discarded db handle) means the reconnect path provides false hope.

### Issue 2: storage:evicted Dead-End Wiring (Significant)

`bus.emit('storage:evicted', {})` fires in glasses-main.ts line 53 when the sentinel is missing but prior data existed. However:
- No `bus.on('storage:evicted', ...)` exists anywhere in the codebase
- Hub only logs `console.warn('[hub] Storage eviction detected')`
- No user notification in either context

The detection works correctly but the signal terminates at the bus with no consumer. Phase 18 will need to wire subscribers.

### Issue 3: Hub Storage Health Gap (Significant)

Hub calls `storageHealth.getQuota()` but only acts on `!quota.isPersisted`. No `persistence:health` event is emitted, no 80%/95% threshold warnings are logged, and the quota result is effectively discarded. Storage pressure in the hub context goes completely unreported.

## Tech Debt

### Phase 14: Data Integrity Foundation

- **reopenDB() stale handle:** Stores not refreshed after reconnection (critical integration gap — affects RES-15)
- **Hub eviction signal:** Only `console.warn()` — no hookable event for Phase 18
- **Hub fatal persistence error:** Only `console.error()` — no user-visible signal
- **Hub requestPersistence() result:** Silent call with no logging of grant/denial
- **storage:evicted dead-end:** Event emitted in glasses but terminates at bus with no subscriber

### Phase 15: Write Verification & Auto-Save Hardening

- **Dual-emit ordering:** `persistence:error` fires before `persistence:warning` on retry exhaustion — Phase 18 error presenter must handle this ordering
- **No error UI consumer:** `persistence:error` type `write-failed`/`verify-failed` emitted but no production UI reacts (Phase 18 dependency)

### Phase 16: Sync Hardening

- **Teardown gap:** `driftReconciler.destroy()` not called in glasses `cleanup()` or hub `beforeunload` — stale mismatch counter on re-init
- **Unused liveness:** `isAlive()` built and tested but never consumed — liveness detection not surfaced
- **Heartbeat-only seq:** RES-09 gap detection tracks heartbeat sequence only — non-heartbeat messages bypass because `seq` is never stamped on `message:added`, `session:switched`, etc.

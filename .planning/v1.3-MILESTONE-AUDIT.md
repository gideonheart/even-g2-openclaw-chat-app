## Cross-Phase Integration

### Current Integration Issues

| # | Finding | Severity | Affected REQs | Status |
|---|---------|----------|---------------|--------|
| 1 | Hub eviction emits `database-closed` type (semantically wrong for eviction) | Low | RES-04, RES-17 | New (Phase 18.5) |
| 2 | `quota-exceeded` type defined and mapped but never emitted at runtime | Low | RES-02, RES-16, RES-17 | Pre-existing |
| 3 | `storage:evicted` bus event emitted but has zero runtime subscribers | Low | RES-04, RES-20 | Pre-existing |
| 4 | Hub error presenter's `fsm:watchdog-reset` subscription unreachable (glasses-only event) | Low | RES-17 | Pre-existing (Phase 18) |
| 5 | `hideBanner` accepted in hub-error-presenter opts but never used | Low | RES-17 | Pre-existing (Phase 18) |

All 5 issues are **low severity**. None block core functionality. The primary error paths all work correctly.

#### Issue 1: Hub Eviction Wrong Type (New — Phase 18.5)

`hub-main.ts` line 1069 emits `persistence:error` with `type: 'database-closed'` for storage eviction. This maps to "Data connection lost" user message instead of an eviction-specific message. Functional (user gets notified) but semantically misleading.

#### Issue 2: quota-exceeded Never Emitted (Pre-existing)

Both boot sequences check `quota.usagePercent >= 95` and call `addLog` but neither emits `persistence:error` with `type: 'quota-exceeded'`. Error presenters have "Storage full" message mappings and tests, but the production path never fires the event.

#### Issue 3: storage:evicted Orphaned Event (Pre-existing)

`glasses-main.ts` emits `bus.emit('storage:evicted', {})` but no subscriber exists. The UX path works via the `evictionDetected` boolean flag and `renderer.showError()`, not via the bus event. Benign but inconsistent.

#### Issue 4: Hub fsm:watchdog-reset Unreachable (Pre-existing)

`hub-error-presenter.ts` subscribes to `fsm:watchdog-reset` on `hubBus`, but the FSM watchdog only fires in the glasses WebView context. No bridge relays this event. By design (hub has no FSM), but the subscription is dead code.

#### Issue 5: hideBanner Unused in Presenter (Pre-existing)

`hideBanner` is passed in opts but never destructured or called. Banner dismiss works via separate DOM event listener in `initHub()`. Presenter cannot auto-dismiss banners when errors resolve.

**Restructure `boot()` in `src/glasses-main.ts`:**

The current `boot()` function creates modules but discards some references (gesture handler and voice loop controller are not stored). Restructure to retain ALL module references and register a cleanup function.

**1. Retain module references:**

Change the existing code so all module creation results are stored in `const` variables. The gesture handler (Layer 3) and voice loop controller (Layer 5) are currently created without storing the return value -- assign them to variables:
```typescript
const gestureHandler = createGestureHandler({ ... });
// ...
const voiceLoopController = createVoiceLoopController({ ... });
```

**2. Add cleanup function with double-call guard:**

After all module initialization (after the gateway health check block), add:
```typescript
let cleaned = false;
function cleanup(): void {
  if (cleaned) return;
  cleaned = true;

  // Reverse initialization order (Layer 5 -> Layer 0)
  voiceLoopController.destroy();
  gateway.destroy();           // stops heartbeat, aborts in-flight fetch
  displayController.destroy(); // stops icon animator, clears flush timer
  gestureHandler.destroy();    // unsubscribes bus listeners
  // audioCapture has no destroy() -- stopRecording is best-effort cleanup
  audioCapture.stopRecording().catch(() => {});
  bridge.destroy();            // unsubscribes SDK, shuts down page container
  bus.clear();                 // clear all remaining subscriptions
}
```

The `cleaned` flag prevents double cleanup when both `visibilitychange` and `pagehide` fire in sequence (Pitfall 4 from research).

**3. Register cleanup handlers (glasses mode only):**

Only register lifecycle cleanup in glasses mode (not devMode). In browser dev mode, tab switching fires `visibilitychange` with `hidden`, which would destroy the voice loop during normal development.

```typescript
if (!devMode) {
  document.addEventListener('visibilitychange', () => {
    if (document.visibilityState === 'hidden') {
      cleanup();
    }
  });
  window.addEventListener('pagehide', cleanup);
}
```

Place this block at the end of `boot()`, after the gateway health check block.

**Important:** Do NOT export the cleanup function. It is internal to boot(). The only way to trigger cleanup is via the browser events.

  <name>Task 1: Create display controller wiring bus events to GlassesRenderer</name>
  <files>
    src/display/display-controller.ts
    src/types.ts
  </files>
  <action>
    Create src/display/display-controller.ts as a factory function:

    ```
    createDisplayController(opts: {
      bus: EventBus&lt;AppEventMap&gt;;
      renderer: GlassesRenderer;
      gestureHandler: GestureHandlerAPI;
    }): DisplayController
    ```

    Export DisplayController interface:
    - init(): Promise&lt;void&gt; -- calls renderer.init(), subscribes to all bus events
    - destroy(): void -- unsubscribes from all bus events, calls renderer.destroy()

    Bus event subscriptions (all stored as unsub functions in an array, cleaned up on destroy):

    1. **Streaming wiring (CHAT-03):**
       - bus.on('gateway:chunk', (chunk) => { ... })
         - If chunk.type === 'transcript': call renderer.addUserMessage(chunk.text!) then renderer.setIconState('sent')
         - If chunk.type === 'response_start': call renderer.startStreaming(), renderer.setIconState('thinking')
         - If chunk.type === 'response_delta': call renderer.appendStreamChunk(chunk.text!)
         - If chunk.type === 'response_end': call renderer.endStreaming(), renderer.setIconState('idle')
         - If chunk.type === 'error': call renderer.endStreaming(), renderer.setIconState('idle')

    2. **Scroll wiring (CHAT-05, CHAT-06):**
       - bus.on('gesture:scroll-up', () => { if (!renderer.isHidden()) renderer.scrollUp() })
       - bus.on('gesture:scroll-down', () => { if (!renderer.isHidden()) renderer.scrollDown() })

    3. **Hide/wake wiring (UIMD-01, UIMD-02):**
       - bus.on('gesture:double-tap', () => {
           if (renderer.isHidden()) { renderer.wake(); }
           // Note: hide is triggered by gesture:menu-toggle when NOT in active conversation
           // The gesture FSM already handles double-tap -> TOGGLE_MENU
         })
       - bus.on('gesture:menu-toggle', ({ active }) => {
           // When menu toggles ON while display is visible, hide the display
           // When menu toggles OFF, wake the display
           if (active) { renderer.hide(); }
           else { renderer.wake(); }
         })

    4. **Icon state wiring:**
       - bus.on('audio:recording-start', () => renderer.setIconState('recording'))
       - bus.on('audio:recording-stop', () => renderer.setIconState('sent'))

    5. **Hint bar wiring:**
       - After each event that might change gesture state, update hint:
         After any gesture event handler, call: renderer.updateHint(gestureHandler.getHintText())
       - To implement this cleanly: subscribe to 'gesture:tap', 'gesture:double-tap', 'gesture:scroll-up', 'gesture:scroll-down' and after each, call renderer.updateHint(gestureHandler.getHintText())
       - NOTE: These hint-update subscriptions must fire AFTER the gesture handler processes the event (so getHintText reflects the new state). Since the event bus dispatches synchronously to all handlers in registration order, register the display controller's hint-update handlers AFTER the gesture handler is created. Document this ordering requirement in a code comment.

    Add to AppEventMap in src/types.ts if not already present from Plan 01:
    - Verify 'display:state-change', 'display:viewport-update', 'display:hide', 'display:wake' exist (added in Plan 01)
    - No additional event types needed

    Internal pattern: collect all unsub functions in an array. On destroy(), call each and clear the array (same pattern as gesture-handler.ts).

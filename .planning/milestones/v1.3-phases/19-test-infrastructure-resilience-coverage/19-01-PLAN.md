Create `src/__tests__/helpers/failure-helpers.ts` with three exported factory functions:

1. **createFailingStore(realStore: ConversationStore, failAfterN: number): ConversationStore**
   - Wraps a real ConversationStore using spread operator
   - Tracks write count across addMessage and createConversation calls
   - After `failAfterN` successful writes, subsequent write calls throw `new DOMException('Simulated write failure', 'AbortError')`
   - All read methods (getConversation, getAllConversations, getMessages, getMessage, getLastConversation, searchMessages, countMessages, verifyMessage) pass through to real store unchanged
   - updateConversation passes through unchanged (not a primary write path for resilience testing)
   - Import ConversationStore type from `../../persistence/types`

2. **createQuotaExceededStore(realStore: ConversationStore): ConversationStore**
   - Wraps a real ConversationStore using spread operator
   - addMessage and createConversation always throw `new DOMException('Quota exceeded', 'QuotaExceededError')`
   - All read methods pass through to real store unchanged

3. **createLossySyncBridge(realBridge: SyncBridge, dropEveryN: number): SyncBridge**
   - Wraps a real SyncBridge
   - Tracks message count on postMessage calls
   - Every Nth message (msgCount % dropEveryN === 0) is silently dropped (not forwarded)
   - onMessage delegates to realBridge.onMessage
   - destroy delegates to realBridge.destroy
   - Import SyncBridge and SyncMessage types from `../../sync/sync-types`

Do NOT add `.test.ts` suffix -- this is a helper module, not a test file. The vite.config.ts `include: ['src/**/*.test.ts']` pattern ensures it is not auto-executed by Vitest.

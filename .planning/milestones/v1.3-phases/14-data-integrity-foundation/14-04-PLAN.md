**Gap being closed:** The `setOnUnexpectedClose` callbacks in both boot files emit `persistence:error` / log the error but never call `reopenDB()`. The `reopenDB()` function is already exported from `db.ts` with retry logic (3 attempts, 1s cooldown). This task wires it in.

**src/glasses-main.ts:**

1. Update the import on line 16 to include `reopenDB`:
```typescript
import { openDB, isIndexedDBAvailable, setOnUnexpectedClose, reopenDB } from './persistence/db';
```

2. Replace the `setOnUnexpectedClose` callback (currently lines 86-93) with:
```typescript
setOnUnexpectedClose(() => {
  bus.emit('persistence:error', {
    type: 'database-closed',
    recoverable: true,
    message: 'Database connection unexpectedly closed',
  });
  bus.emit('log', { level: 'error', msg: 'Database connection unexpectedly closed' });

  // Attempt to reopen the database (RES-15)
  reopenDB().then((newDb) => {
    bus.emit('log', { level: 'info', msg: 'Database reconnected successfully' });
  }).catch(() => {
    bus.emit('persistence:error', {
      type: 'database-closed',
      recoverable: false,
      message: 'Failed to reopen database after max retries',
    });
    bus.emit('log', { level: 'error', msg: 'Database reopen failed after max retries -- restart required' });
  });
});
```

The `.then()/.catch()` pattern (not async/await) is intentional: the onclose callback is synchronous and fire-and-forget. The reopenDB Promise runs in the background. On success, the new db is returned by reopenDB but not consumed here -- the new connection is held inside db.ts module state via the openDB() call within reopenDB(). On failure (max retries exhausted), a `recoverable: false` error is emitted so downstream error UX (Phase 18) can show a "restart required" message.

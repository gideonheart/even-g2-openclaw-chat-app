**Step 1: Create `src/display/error-presenter.ts`**

Implement `createGlassesErrorPresenter` factory using the unsub-array pattern (same as display-controller.ts).

Interface:
```typescript
export interface GlassesErrorPresenter {
  destroy(): void;
}

export function createGlassesErrorPresenter(opts: {
  bus: EventBus<AppEventMap>;
  bridge: BridgeService;
  renderer: GlassesRenderer;
  iconAnimator: { stop(): void; start(): void };
}): GlassesErrorPresenter
```

Internal state:
- `const unsubs: Array<() => void> = [];`
- `let statusClearTimer: ReturnType<typeof setTimeout> | null = null;`
- `let lastErrorLevel: 'warning' | 'error' | null = null;`

`showStatusBar(message: string, durationMs: number)`:
1. If `statusClearTimer` is set, `clearTimeout(statusClearTimer)`.
2. Call `opts.iconAnimator.stop()` to pause animation.
3. Call `opts.bridge.textContainerUpgrade(1, message)`.
4. Set `statusClearTimer = setTimeout(() => { statusClearTimer = null; lastErrorLevel = null; opts.iconAnimator.start(); }, durationMs)`.

`mapErrorTypeToMessage(type: string): string`:
- `'database-closed'` → `'Data connection lost — reconnecting'`
- `'quota-exceeded'` → `'Storage full'`
- `'write-failed'` → `'Could not save message'`
- `'verify-failed'` → `'Storage check failed'`
- default → `'Something went wrong'`

Bus subscriptions (push all unsubs):
1. `bus.on('persistence:error', ({ type, recoverable }) => {`:
   - If `lastErrorLevel === 'error'` return (already showing worse error — Pitfall 2 guard).
   - `lastErrorLevel = 'error';`
   - `const userMsg = mapErrorTypeToMessage(type);`
   - Duration: recoverable=true → 10_000ms with suffix `' — tap to retry'`, recoverable=false → 3_000ms.
   - Call `showStatusBar(userMsg + suffix, durationMs)`.
2. `bus.on('fsm:watchdog-reset', ({ previousState }) => {`:
   - Call `showStatusBar(`${previousState} timed out — tap to retry`, 5_000)`.

`destroy()`:
- `if (statusClearTimer) clearTimeout(statusClearTimer);`
- For each unsub, call it.
- `unsubs.length = 0;`

Imports needed: `EventBus` from `'../events'`, `AppEventMap` from `'../types'`, `BridgeService` from `'../bridge/bridge-types'`, `GlassesRenderer` from `'./glasses-renderer'`.

- **Option A (PREFERRED):** Pass a minimal `iconAnimator`-shaped object using `renderer`'s `setIconState` method isn't sufficient. Instead, expose `iconAnimator` from `GlassesRenderer` interface as a getter. Add `getIconAnimator(): { stop(): void; start(): void } | null` to the `GlassesRenderer` interface in `glasses-renderer.ts` and return `iconAnimator` from it.
- **Option B (FALLBACK):** Inline stop/start logic via `bridge.textContainerUpgrade(1, '')` to simulate stop. Do NOT use this — it doesn't actually stop the timer.

Use Option A: add `getIconAnimator(): { stop(): void; start(): void } | null` to GlassesRenderer interface and implementation in `glasses-renderer.ts`. Then:

  bus,

Add `glassesErrorPresenter.destroy()` to the cleanup function (before `displayController.destroy()`).

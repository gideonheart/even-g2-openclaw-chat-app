**Fix 1: reopenDB handle propagation (RES-15, Critical)**

In the setOnUnexpectedClose callback (currently lines 124-143), replace the reopenDB success handler that discards the new db handle. The current code is:
```
reopenDB().then(() => {
  bus.emit('log', { level: 'info', msg: 'Database reconnected successfully' });
}).catch(...)
```

Replace with code that:
1. Accepts `newDb` parameter from `reopenDB().then((newDb) => ...)`
2. Reassigns `store = createConversationStore(newDb)` and `sessionStore = createSessionStore(newDb, store)`
3. Destroys old autoSave and creates a new one with identical config (bus, new store, getConversationId getter, onConversationNamed callback, syncBridge) -- this is critical because autoSave captures store in closure at creation time
4. Destroys old driftReconciler and creates a new one with new store and same onDriftDetected/onReconciled callbacks
5. Destroys old syncMonitor and creates a new one with new store, same bridge, same origin/getActiveConversationId, and onHeartbeat wired to new driftReconciler
6. Calls syncMonitor.startHeartbeat() on the new instance
7. Emits log: 'Database reconnected -- all stores refreshed'

Key details:
- The `store`, `sessionStore`, `autoSave`, `driftReconciler`, `syncMonitor` variables must be declared with `let` (not `const`) so they can be reassigned. Check that `autoSave` at line 354 is `const` and change to `let`. Same for `driftReconciler` (line 157) and `syncMonitor` (line 172) -- these are already using ternary expressions assigned to `const`, change to `let`.
- The callback closures for autoSave.onConversationNamed and syncMonitor.onHeartbeat reference `activeConversationId` and `syncBridge` which are stable (not db-dependent), so they can be reused as-is.
- Do NOT recreate syncBridge (it is IDB-independent, uses BroadcastChannel/localStorage).
- Do NOT recreate sessionManager or menuController (these are Layer 4b, can continue with stale session data until next user action refreshes them).

**Fix 2: storage:evicted subscriber (RES-04, Significant)**

Add a flag `let evictionDetected = false;` near the top of boot(), before the IDB init block. Inside the sentinel check block (around line 52-53) where `bus.emit('storage:evicted', {})` is called, also set `evictionDetected = true`.

After the renderer init and message restore section (after line 246, after the welcome/restore messages), add:
```typescript
if (evictionDetected) {
  renderer.showError('Data was cleared by system');
}
```

This uses the flag pattern from the research because the event fires during boot before the renderer exists. Do NOT use bus.on('storage:evicted') since the emit fires synchronously before any subscriber could be registered.

**Fix 3: driftReconciler cleanup (RES-11, Minor)**

In the cleanup() function (line 401-418), add `driftReconciler?.destroy();` as the FIRST line after the guard check, BEFORE `syncMonitor?.destroy()`. This clears the mismatch counter before stopping the heartbeat.

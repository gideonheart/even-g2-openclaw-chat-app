Make four targeted changes to src/hub-main.ts:

**Fix 1: reopenDB handle propagation (RES-15, Critical)**

In initPersistence(), the setOnUnexpectedClose callback (currently lines 1039-1048) discards the reopenDB result. Replace:
```
reopenDB().then(() => {
  console.log('[hub] Database reconnected successfully');
}).catch(...)
```

With code that:
1. Accepts `newDb` from `reopenDB().then((newDb) => ...)`
2. Creates new stores: `const newConversationStore = createConversationStore(newDb)` and `const newSessionStore = createSessionStore(newDb, newConversationStore)`
3. Updates module-level `hubConversationStore = newConversationStore`
4. Calls `loadLiveConversation()` to reload the conversation view from the fresh store
5. Logs success: `addLog('info', 'Database reconnected -- stores refreshed')`
6. On failure: `addLog('error', 'Database reopen failed -- restart required')` + `showToast('Database error -- please restart the app')`

Key details:
- Hub is simpler than glasses because `sessionManager`, `driftReconciler`, and `syncMonitor` are created inside `initPersistence()` and we do NOT need to recreate all of them. The `hubConversationStore` module-level variable is the critical one -- it's used by `loadLiveConversation()`, `handleHubChunk()`, `handleTextSubmit()`, and all chat/search functions.
- `loadLiveConversation()` reads from `hubConversationStore`, so updating the module-level variable before calling it ensures it uses the fresh store.
- Do NOT try to recreate sessionManager (would require re-wiring all sync message handlers -- too complex for gap closure).

**Fix 2: Hub storage health emission (RES-02, Significant)**

In initPersistence(), the storage health section (currently lines 1031-1036) only checks isPersisted. Replace:
```typescript
const storageHealth = createStorageHealth();
const quota = await storageHealth.getQuota();
if (quota.isAvailable && !quota.isPersisted) {
  await storageHealth.requestPersistence();
}
```

With:
```typescript
const storageHealth = createStorageHealth();
const quota = await storageHealth.getQuota();
if (quota.isAvailable) {
  if (quota.usagePercent >= 95) {
    addLog('error', `Storage critical: ${quota.usagePercent.toFixed(1)}% used`);
  } else if (quota.usagePercent >= 80) {
    addLog('warn', `Storage warning: ${quota.usagePercent.toFixed(1)}% used`);
  }
  if (!quota.isPersisted) {
    const granted = await storageHealth.requestPersistence();
    addLog('info', `Persistent storage ${granted ? 'granted' : 'denied'}`);
  }
}
```

This matches the glasses-main.ts pattern (lines 108-121). Use `addLog()` not event bus -- hub has no EventBus (per Phase 14 decision, deferred to Phase 18).

**Fix 3: Hub eviction notification (RES-04, Significant)**

In initPersistence(), after the sentinel check (around line 988-989), when eviction is detected, the current code only does `console.warn('[hub] Storage eviction detected')`. Add two lines after that console.warn:
```typescript
addLog('error', 'Storage eviction detected -- previous conversations were lost');
showToast('Previous data was cleared by system');
```

This uses the hub's existing notification patterns (addLog + showToast) rather than introducing an event bus.

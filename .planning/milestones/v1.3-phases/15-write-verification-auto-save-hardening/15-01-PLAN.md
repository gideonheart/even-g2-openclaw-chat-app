<task type="auto">
  <name>Task 2: Enhance auto-save with verification, escalation, and partial save</name>
  <files>src/persistence/auto-save.ts</files>
  <action>
Modify `createAutoSave()` in `src/persistence/auto-save.ts` to implement all three RES requirements:

**RES-06: First-write verification**
1. Add `let storageVerified = false;` state variable alongside existing `pendingAssistantText` and `hasUserMessage`.
2. Add an async function `verifyFirstWrite(messageId: string): Promise<void>`:
   - If `storageVerified` is true, return immediately (skip verification)
   - Call `store.verifyMessage(messageId)`
   - If returns true: set `storageVerified = true`
   - If returns false: emit `bus.emit('persistence:error', { type: 'verify-failed', recoverable: false, message: 'Storage verification failed -- first message not readable', conversationId: getConversationId() })`
   - Do NOT set storageVerified on failure (allow re-verification) per Pitfall P5
3. In the `transcript` case, capture the message ID from `addMessage()`. Use closure variable pattern:
   - Declare `let lastSavedId = ''` before `saveWithRetry()`
   - Wrap the operation: `async () => { lastSavedId = await store.addMessage(convId, {...}); }`
   - In `.then((ok) => { if (ok && !storageVerified) { verifyFirstWrite(lastSavedId); } ... })`
4. Add a `persistence:warning` listener that resets `storageVerified = false`:
   ```
   unsubs.push(bus.on('persistence:warning', () => { storageVerified = false; }));
   ```

**RES-07: Error escalation in saveWithRetry**
1. Modify `saveWithRetry()` signature to accept optional context: `saveWithRetry(operation: () => Promise<unknown>, context?: { conversationId: string }): Promise<boolean>`
2. After the retry loop exhausts (before `return false`), emit:
   ```
   bus.emit('persistence:error', {
     type: 'write-failed',
     recoverable: false,
     message: 'Failed to save message after retries',
     conversationId: context?.conversationId,
   });
   ```
3. Update ALL existing `saveWithRetry()` call sites to pass `{ conversationId: convId }` as second argument.
4. KEEP the existing `.then(!ok)` blocks that emit `persistence:warning` -- this is intentional dual-emit per Pitfall P2 (warning for soft handler, error for Phase 18 error presenter).

**RES-08: Partial response preservation on error**
1. Replace the `case 'error'` block. Instead of:
   ```
   pendingAssistantText = '';
   ```
   Do:
   ```
   if (pendingAssistantText) {
     const convId = getConversationId();
     const text = pendingAssistantText + ' [response interrupted]';
     pendingAssistantText = '';
     saveWithRetry(
       () => store.addMessage(convId, { role: 'assistant', text, timestamp: Date.now() }),
       { conversationId: convId },
     ).then((ok) => {
       if (ok && syncBridge) {
         syncBridge.postMessage({
           type: 'message:added', origin: 'glasses',
           conversationId: convId, role: 'assistant', text,
         });
       }
     });
   } else {
     pendingAssistantText = '';
   }
   ```
2. Keep the existing `streaming:end` sync bridge posting after the partial save block (unchanged).

Key constraints:
- Do NOT verify every write (Pitfall P1) -- only first write per session
- Do NOT block subsequent saves on verification failure (Pitfall P5)
- Do NOT remove existing persistence:warning emissions (backward compatibility)
- Verification is async/fire-and-forget -- does not block the save flow
- The "[response interrupted]" suffix is hardcoded English (per research Open Question 3)
  </action>

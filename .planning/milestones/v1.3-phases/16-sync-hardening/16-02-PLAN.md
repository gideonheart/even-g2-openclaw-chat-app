1. Create DriftReconciler (needs store, bus):
```typescript
const driftReconciler = store ? createDriftReconciler({
  store,
  onDriftDetected: (info) => {
    bus.emit('sync:drift-detected', info);
    bus.emit('log', {
      level: 'warn',
      msg: `Sync drift: local=${info.localCount} remote=${info.remoteCount} conv=${info.conversationId}`,
    });
  },
  onReconciled: (info) => {
    bus.emit('sync:reconciled', info);
    bus.emit('log', { level: 'info', msg: `Sync reconciled: ${info.conversationId}` });
  },
}) : null;
```

2. Create SyncMonitor wrapping the existing syncBridge:
```typescript
const syncMonitor = store ? createSyncMonitor({
  bridge: syncBridge,
  store,
  origin: 'glasses',
  getActiveConversationId: () => activeConversationId,
  onHeartbeat: driftReconciler
    ? (conversationId, remoteCount) => {
        driftReconciler.handleHeartbeat(conversationId, remoteCount);
      }
    : undefined,
}) : null;
```

3. Wire reconciliation to re-render. Add a `sync:reconciled` bus listener **after the switchToSession function definition** (around line ~244):
```typescript
// Sync drift reconciliation: re-read from IDB and re-render when drift detected
bus.on('sync:reconciled', async ({ conversationId }) => {
  if (conversationId === activeConversationId && store) {
    renderer.destroy();
    await renderer.init();
    const messages = await store.getMessages(conversationId);
    for (const msg of messages) {
      if (msg.role === 'user') {
        renderer.addUserMessage(msg.text);
      } else {
        renderer.startStreaming();
        renderer.appendStreamChunk(msg.text);
        renderer.endStreaming();
      }
    }
  }
});
```

4. Start heartbeat **after display init and restore are complete** (after the persistence:restored / welcome message block, around line ~217). Add:
```typescript
syncMonitor?.startHeartbeat();
```

5. Update cleanup function: add `syncMonitor?.destroy()` BEFORE `syncBridge.destroy()` in the cleanup sequence:
```typescript
syncMonitor?.destroy();   // stop heartbeat before bridge teardown
syncBridge.destroy();     // cross-context sync (existing)
```

**IMPORTANT:** Do NOT change how auto-save or session manager call syncBridge.postMessage(). Per research Open Question 3, SyncMonitor.send() is separate from the bridge -- SyncMonitor is an observer that adds seq numbers to its own heartbeat messages. Existing callers continue using syncBridge.postMessage() directly. The SyncMonitor intercepts incoming messages via onMessage subscription for tracking.

This means existing auto-save and session code is UNCHANGED. SyncMonitor only adds seq to messages it sends (heartbeat), and observes all incoming messages for sequence tracking.

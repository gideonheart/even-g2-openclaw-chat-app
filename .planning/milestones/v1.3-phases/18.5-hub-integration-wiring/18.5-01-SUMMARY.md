## Files Created/Modified

## Decisions Made
- Hub error paths replace direct `showToast()` with `hubBus.emit('persistence:error')` -- the hub error presenter's callback IS the `showToast`, so emitting and calling showToast would produce double toasts
- `loadLiveConversation()` moved to last call in reopenDB success handler (after sync module recreation) to ensure fresh modules are in place before re-rendering
- `cachedQuota` stored as module-level variable so `refreshHealthDisplay()` can remain synchronous -- storage quota doesn't change rapidly enough within a session to require real-time re-reads
- `persistence:health` emitted after the `hubBus.on('persistence:health', ...)` subscriber registration to guarantee the subscriber is in place before the initial quota fires

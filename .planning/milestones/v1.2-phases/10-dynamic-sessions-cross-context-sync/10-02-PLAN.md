```typescript
// Handle sync messages from hub context
syncBridge.onMessage((msg) => {
  if (msg.origin === 'glasses') return; // ignore own echoes

  switch (msg.type) {
    case 'session:switched': {
      // Hub switched session -- load new session into display
      switchToSession(msg.sessionId);
      break;
    }
    case 'session:deleted': {
      // Hub deleted a session -- if it was active, switch to most recent
      if (msg.sessionId === activeConversationId && sessionStore) {
        sessionStore.listSessions().then((sessions) => {
          if (sessions.length > 0) {
            switchToSession(sessions[0].id);
          }
          // If no sessions remain, a new one will be created on next voice turn
        });
      }
      break;
    }
    case 'session:created':
    case 'session:renamed':
      // No glasses-side action needed -- hub UI handles these
      break;
  }
});
```

**5. Create a `switchToSession` helper function inside boot():**

```typescript
async function switchToSession(sessionId: string): Promise<void> {
  const previousId = activeConversationId;
  activeConversationId = sessionId;
  writeActiveConversationId(sessionId);

  // Clear display and load new session's messages
  renderer.destroy();
  await renderer.init();

  if (store) {
    const messages = await store.getMessages(sessionId);
    for (const msg of messages) {
      if (msg.role === 'user') {
        renderer.addUserMessage(msg.text);
      } else {
        renderer.startStreaming();
        renderer.appendStreamChunk(msg.text);
        renderer.endStreaming();
      }
    }
  }

  // Emit local bus event for interested modules (auto-save uses getConversationId getter)
  bus.emit('session:switched', { id: sessionId, previousId });
}
```

**IMPORTANT considerations per research pitfall #5 (session switch during streaming):**
- The `autoSave` module's `getConversationId` getter already returns `activeConversationId` by reference, so updating `activeConversationId` immediately redirects new saves to the correct session.
- The `pendingAssistantText` in auto-save accumulates response_delta chunks. If a switch happens mid-stream, the response_end will save to the new session -- which is incorrect. To handle this: reset the display controller's streaming state by calling `renderer.endStreaming()` before switching if a stream is active. The auto-save's pending text will be discarded when the `response_end` fires for the old turn (since the conversationId changed, the orphaned save is a minor acceptable tradeoff per research).

**6. Add syncBridge to cleanup:**
In the `cleanup()` function, add `syncBridge.destroy();` before `autoSave?.destroy()` (since syncBridge has no dependencies on other modules).

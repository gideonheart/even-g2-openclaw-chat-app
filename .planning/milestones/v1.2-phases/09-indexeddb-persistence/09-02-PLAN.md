**Create `src/persistence/auto-save.ts`:**

Export `createAutoSave(opts)` factory function:

```typescript
interface AutoSaveOptions {
  bus: EventBus<AppEventMap>;
  store: ConversationStore;
  getConversationId: () => string;
  /** Called when auto-naming triggers on first user message */
  onConversationNamed?: (name: string) => void;
}
```

Implementation:
- Subscribe to `gateway:chunk` on the event bus
- Track state: `pendingAssistantText = ''` and `isFirstMessage` flag per conversation
- On `transcript` chunk:
  - Call `store.addMessage(getConversationId(), { role: 'user', text: chunk.text!, timestamp: Date.now() })`
  - If this is the first user message in the conversation (no messages existed before), call `generateConversationName(chunk.text!)` and update the conversation name via `store.updateConversation()`. Invoke `onConversationNamed` callback if provided.
  - Fire-and-forget: wrap in `.catch()` with retry logic (max 3 retries, 500ms backoff)
  - If all retries fail, emit `bus.emit('persistence:warning', { message: 'Messages may not be saved' })`
- On `response_delta`: accumulate text in `pendingAssistantText`
- On `response_end`: if `pendingAssistantText` is non-empty, save as assistant message (same retry logic). Reset `pendingAssistantText = ''`.
- On `error` chunk: reset `pendingAssistantText = ''` (don't save failed responses)
- Return `{ destroy() }` that unsubscribes from bus

**IMPORTANT per user decision:** Only save assistant messages on `response_end`, NOT during streaming. Save is completely invisible -- no UI feedback on success.

**Create `src/persistence/boot-restore.ts`:**

Export `restoreOrCreateConversation(opts)`:

```typescript
interface RestoreOptions {
  store: ConversationStore | null;  // null = IndexedDB unavailable
}

interface RestoreResult {
  conversationId: string;
  messages: MessageRecord[];
  restored: boolean;             // true if loaded from DB, false if fresh
  storageAvailable: boolean;     // false = in-memory only mode
  error?: string;                // set if corruption detected
}
```

Implementation:
- If `store` is null: return `{ conversationId: crypto.randomUUID(), messages: [], restored: false, storageAvailable: false }`
- Try `store.getLastConversation()`:
  - If found: load messages via `store.getMessages(conv.id)`. Return with `restored: true`.
  - If not found: create new conversation via `store.createConversation()`. Return with `restored: false`.
- If any error during restore: return fresh conversation with `error: "Previous conversation couldn't be restored"`.
- Also store/read active conversation ID in localStorage key `'openclaw-active-conversation'` for fast boot.
